Requisitos de Información

RI.1. Información de Pacientes
La aplicación deberá almacenar la información sobre los pacientes que utilice dicho servicio, en concreto:
 Nombre y apellidos
 NUHSA
 Peso
 Altura
 Fecha de nacimiento
 Alergias
 Actividad física diaria
 IMC inicial
 Ingesta calórica diaria

RI.2. Información sobre alimentos
La aplicación deberá almacenar la información sobre los alimentos relacionados con la dieta mediterránea, en concreto:
 Nombre
 Kilocalorías
 Hidratos de carbono
 Grasas Saturadas
 Grasas Insaturadas
 Proteínas
 Consumo semanal recomendado

RI.3. Información sobre dietas
La aplicación deberá almacenar la información sobre las dietas propuestas por el médico para cada paciente, en concreto:
 Paciente
 Tabla de fraccionamiento de ingesta calórica correspondiente a cada comida (Desayuno, Media Mañana, Almuerzo, Merienda y Cena).
 Lista de alimentos permitidos para cada comida.
 Duración
 Indicaciones médicas

RI.4. Información sobre médicos
La aplicación deberá almacenar la información sobre los médicos que asignen las dietas a cada paciente, en concreto:
 Nombre
 Especialidad
 Centro
 Consulta
 Email

RI.5. Información sobre citas
La aplicación deberá almacenar la información sobre las citas, en concreto:
 Fecha
 Hora
 Medico
 Paciente

Requisitos Funcionales
 RF1: La aplicación permitirá añadir nuevos pacientes.
 RF2: La aplicación permitirá editar la información de los pacientes registrados.
 RF3: La aplicación permitirá eliminar los pacientes registrados
 RF4: La aplicación permitirá listar los pacientes registrados.
 RF5: La aplicación permitirá calcular el IMC a través del peso y la altura del paciente mediante la siguiente expresión:
  o IMC= masa/altura^2
 RF6: La aplicación permitirá listar las alergias del paciente
 RF7: La aplicación permitirá listar los alimentos.
 RF8: La aplicación permitirá al médico modificar la ingesta calórica del paciente.
 RF9: La aplicación permitirá al médico crear la tabla de fraccionamiento de ingesta calórica correspondiente a cada de las 5 comidas diarias recomendadas.
 RF10: La aplicación permitirá al médico editar la tabla de fraccionamiento de ingesta calórica por comidas.
 RF11: La aplicación permitirá al médico crear la lista de alimentos permitidos para cada comida.
 RF12: La aplicación permitirá al médico editar la lista de alimentos permitidos para cada comida.
 RF13: La aplicación permitirá relacionar las dietas con los pacientes.
 RF14: La aplicación ofrecerá información sobre las dietas.
 RF15: La aplicación permitirá crear citas.
 RF16: La aplicación permitirá editar citas.
 RF17: La aplicación permitirá cancelar citas.
 RF18: La aplicación permitirá conocer los alimentos más consumidos durante las dietas.
